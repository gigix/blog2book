Interesting Thread on RoR- -

                                      

Michael Yuan: Is Ruby replacing Java? Not so fast!

The question regarding to Ruby versus Java is NOT how much more advanced RoR is compared with Java EE -- the real question is whether Java EE is good enough for most developers.

Obie's response: Unwitting Ruby Evangelist

Totally subversive move. I mean, is this Michael Yuan guy secretly a Ruby evangelist or something? Sure, he claims to doubt that Ruby will catch on in the enterprise, but between the lines of his comments, the message rings true: Ruby is superior to what's out there now.

(BTW, don't you know our Planet TW yet?)

And DHH's response: Why Java (presumably) won't go anywhere soon

If you agree to those assumptions, Michael is absolutely right. And lots of software shops, managers, and apparently some programmers do. Not Rails, not any other innovation, evolution, or revolution in software is going to change that any time soon. This is ingrained baggage.

Aha, he's talking about baggage. You BJUG guys, don't you think it's laughable?
